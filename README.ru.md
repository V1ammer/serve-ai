# Serve-AI Benchmark

Набор инструментов для тестирования производительности (бенчмаркинга) локальных ИИ-моделей, адаптированный под аппаратное обеспечение. Специально разработан для запуска моделей Qwen 3.6 (27B Dense и 35B MoE) на системах с ограниченными ресурсами.

## Поддерживаемые движки
- **llama.cpp:** Высокоэффективный вывод на CPU/Vulkan (через `llama-cpp-python`).
- **vLLM / SGLang:** Базовая поддержка для CPU/ROCm бэкендов (требует ручной настройки окружения для AMD APU).

## Установка

### NixOS (Рекомендуется)
Проект включает `flake.nix` и `.envrc` для создания детерминированного окружения.
```bash
direnv allow
uv sync
```

### Ubuntu
Убедитесь, что установлены инструменты сборки:
```bash
sudo apt update && sudo apt install build-essential cmake
uv sync
```

### Windows
1. Установите [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/) (выберите рабочую нагрузку "Разработка классических приложений на C++").
2. Установите `uv`.
3. Выполните `uv sync`.

## Использование
1. **Поиск лучших моделей для вашего железа:**
   ```bash
   python benchmark.py --recommend
   ```
   (Эта команда покажет, какие версии Qwen 3.6 поместятся в вашу оперативную память, и даст ссылки на скачивание).

2. **Подготовка моделей:** Поместите ваши `.gguf` файлы в директорию `models/`.
2. **Запуск стандартного теста:**
   ```bash
   python benchmark.py
   ```
3. **Запуск теста качества кодинга:**
   ```bash
   python benchmark.py --coding
   ```
4. **Запуск модели для локального чата:**
   ```bash
   python serve.py
   ```
   (Следуйте интерактивным подсказкам для выбора модели).

## Структура проекта
- `benchmark.py`: Основная точка входа и отчетность.
- `hardware.py`: Кроссплатформенное определение характеристик железа.
- `engines.py`: Обертки для интеграции llama.cpp и других серверов.
- `models.py`: Конфигурация вариантов Qwen 3.6 и промпты для тестирования кода.
- `flake.nix`: Окружение разработки для NixOS.
